Emuladores de Rede: o que são e como usar EVE-NG, GNS3 e Packet Tracer

Publicado em: 09 de Junho de 2026 • Por: Rinaldo Purger • Infraestrutura e Redes

Montar um laboratório físico de redes com roteadores Cisco, switches gerenciáveis e firewalls sempre foi um privilégio caro — o equipamento necessário para estudar para certificações como CCNA, CCNP ou CompTIA Network+ pode custar vários milhares de reais. Os emuladores de rede resolveram esse problema: hoje você consegue montar topologias complexas com dezenas de dispositivos virtuais no seu próprio computador, gratuitamente.

Neste guia você vai entender o que são emuladores de rede, a diferença entre emulação e simulação, e as características de cada uma das três principais ferramentas do mercado: EVE-NG, GNS3 e Cisco Packet Tracer.

Emulação vs Simulação: qual a diferença?

Antes de falar das ferramentas, é importante entender essa distinção que faz toda a diferença na prática:

Simulação imita o comportamento de um dispositivo por software — como o Cisco Packet Tracer faz. O programa simula como um roteador responderia a determinados comandos, mas não executa o sistema operacional real do equipamento. É uma abstração educacional.

Emulação executa o sistema operacional real do dispositivo (como o Cisco IOS, Juniper JunOS ou Arista EOS) dentro de uma máquina virtual. O EVE-NG e o GNS3 são emuladores — eles rodam o IOS original da Cisco, então o comportamento é idêntico ao do hardware real.

Para estudos iniciais e certificações introdutórias, simulação é suficiente. Para ambientes que precisam de comportamento real — testes de configuração de produção, protocolos avançados, troubleshooting — a emulação é indispensável.

EVE-NG: o favorito dos profissionais

O EVE-NG (Emulated Virtual Environment - Next Generation) é hoje a ferramenta mais usada por profissionais de redes e instrutores de certificações avançadas. Ele roda como uma appliance Linux (baseada em Ubuntu) e é acessado inteiramente pelo navegador — você não precisa instalar nenhum cliente pesado no seu computador.

Como o EVE-NG funciona

O EVE-NG usa tecnologias de virtualização como QEMU, IOU (IOS on Unix) e Docker para rodar múltiplos sistemas operacionais de rede simultaneamente. Você importa as imagens dos dispositivos (IOS da Cisco, JunOS da Juniper, VyOS, pfSense, etc.) e o EVE-NG as executa como máquinas virtuais interconectadas por redes virtuais.

A interface web permite arrastar e soltar dispositivos na topologia, conectar interfaces com um clique e acessar o console de cada dispositivo diretamente pelo navegador via telnet ou SSH. O resultado é uma experiência visual muito próxima do que você teria em uma ferramenta de diagramação como o Visio, mas totalmente funcional.

Requisitos de hardware para EVE-NG

EVE-NG é exigente em hardware — afinal, você estará rodando múltiplos sistemas operacionais simultaneamente:

  • CPU: Intel com suporte a VT-x ou AMD com AMD-V (virtualização habilitada na BIOS). Quanto mais núcleos, melhor — 4 núcleos físicos é o mínimo recomendado
  • RAM: mínimo 8GB, recomendado 16GB ou mais. Cada roteador virtual consome 512MB a 1GB de RAM
  • Armazenamento: SSD com pelo menos 100GB livres. As imagens de dispositivos ocupam bastante espaço
  • Instalação: como VM no VMware Workstation ou VirtualBox, ou bare-metal em um servidor dedicado

GNS3: o veterano open source

O GNS3 (Graphical Network Simulator-3) é o pioneiro dos emuladores de rede e existe desde 2008. Assim como o EVE-NG, ele executa imagens reais de sistemas operacionais de rede. A principal diferença é que o GNS3 tem um cliente desktop (Windows, Mac, Linux) e um servidor separado — o GNS3 VM — que roda as imagens dos dispositivos.

Arquitetura do GNS3

Quando você instala o GNS3, na prática são dois componentes:

  1. GNS3 GUI — o cliente visual que roda no seu Windows/Mac/Linux. Você usa para montar a topologia
  2. GNS3 VM — uma máquina virtual Linux que executa os dispositivos de rede emulados. Recomendado para performance estável

Você pode rodar os dispositivos localmente sem a VM, mas para topologias maiores com Cisco IOS, o GNS3 VM oferece muito mais estabilidade e desempenho.

Pontos fortes do GNS3

  • 100% gratuito e open source — sem versão paga com recursos bloqueados
  • Comunidade enorme — tutoriais, topologias prontas e suporte nos fóruns
  • Integração com Wireshark — capture pacotes em qualquer link da topologia com um clique
  • Suporte a Docker — adicione containers Linux à topologia para simular servidores
  • Marketplace — repositório oficial com appliances prontas para importar

Cisco Packet Tracer: o simulador do iniciante

O Packet Tracer é desenvolvido pela própria Cisco e distribuído gratuitamente para estudantes do Cisco Networking Academy (NetAcad). Diferente dos dois anteriores, ele é um simulador — não executa o IOS real, mas simula o comportamento dos dispositivos Cisco de forma suficientemente fiel para aprender os fundamentos.

Quando usar o Packet Tracer

O Packet Tracer é ideal para quem está começando do zero. Ele roda em qualquer computador (inclusive máquinas modestas), não requer imagens de dispositivos e tem uma interface extremamente intuitiva com atividades guiadas. Para quem está estudando para o CCNA pela primeira vez, é a ferramenta certa para os primeiros meses.

Suas limitações ficam evidentes quando você avança: ele não suporta todos os recursos do IOS real, não tem integração com o Wireshark e não permite emular dispositivos de outros fabricantes (Juniper, Arista, Fortinet).

Instalando o EVE-NG Community no VMware

O método mais simples para começar com o EVE-NG é instalar a appliance OVA dentro do VMware Workstation Player (gratuito). Veja o passo a passo:

  1. Baixe o VMware Workstation Player em vmware.com (gratuito para uso não-comercial)
  2. Acesse eve-ng.net e baixe a OVA da versão Community
  3. No VMware, clique em Open a Virtual Machine e selecione o arquivo OVA
  4. Defina pelo menos 4GB de RAM e 2 CPUs para a VM — mais é melhor
  5. Nas configurações da VM, em Processors, marque Virtualize Intel VT-x/EPT — essencial para nested virtualization
  6. Inicie a VM e aguarde o boot. Anote o IP que aparecer na tela
  7. Acesse esse IP no navegador — a interface web do EVE-NG estará disponível
  8. Login padrão: usuário admin, senha eve
# Após o primeiro login, configure a senha root via SSH
ssh root@IP_DO_EVE # senha padrão: eve
passwd # troque a senha imediatamente

Adicionando imagens de dispositivos ao EVE-NG

O EVE-NG não vem com imagens de dispositivos por questões de licença — você precisa obtê-las separadamente. Para imagens Cisco, você pode extraí-las de equipamentos que já possui licença ou encontrá-las em repositórios da comunidade. As imagens mais usadas são:

  • Cisco IOSv — roteador virtual, leve e estável, ideal para CCNA/CCNP
  • Cisco IOSvL2 — switch virtual Layer 2/3
  • Cisco CSR1000v — roteador cloud, suporta recursos mais avançados
  • Cisco ASAv — firewall ASA virtual
  • VyOS — roteador open source totalmente gratuito e legal para usar
  • pfSense — firewall open source, imagem disponível oficialmente
  • MikroTik RouterOS — imagem CHR gratuita para laboratório

Para carregar uma imagem no EVE-NG, faça upload via SCP ou SFTP para a pasta correta dentro da appliance:

# Copiando imagem IOSv para o EVE-NG via SCP
scp vios-adventerprisek9-m.vmdk.SPA.157-3.M3.zip root@IP_EVE:/opt/unetlab/addons/qemu/

# Após o upload, corrigir permissões
ssh root@IP_EVE
/opt/unetlab/wrappers/unl_wrapper -a fixpermissions

Sua primeira topologia no EVE-NG

Com o EVE-NG instalado e as imagens carregadas, criar uma topologia é intuitivo:

  1. Na interface web, clique em Add new lab
  2. Dê um nome ao laboratório (ex: "OSPF Lab 1")
  3. Clique com o botão direito na área em branco → Add node
  4. Selecione o tipo de dispositivo (ex: Cisco IOSv) e configure nome e RAM
  5. Repita para quantos dispositivos precisar
  6. Para conectar: clique no ícone de cabo no menu → clique no dispositivo de origem → selecione a interface → clique no destino
  7. Clique em Start all nodes no menu
  8. Clique em qualquer dispositivo para abrir o console no navegador

Alternativas baseadas em nuvem

Se seu hardware não suporta rodar um emulador local, existem opções baseadas em nuvem onde você acessa labs de rede pelo navegador sem instalar nada:

  • Cisco DevNet Sandbox — ambientes Cisco gratuitos para desenvolvimento e testes, com reserva por horas em devnetsandbox.cisco.com
  • Cisco Modeling Labs (CML) — versão paga e online do emulador oficial da Cisco, com planos a partir de US$199/ano
  • Juniper vLabs — acesso gratuito a equipamentos Juniper virtuais para estudo do JunOS
  • NetDevOps Labs — laboratórios focados em automação de redes com Ansible e Python

EVE-NG ou GNS3: qual escolher em 2026?

Para a maioria dos profissionais que estão estudando para certificações Cisco ou construindo labs de redes corporativas, a recomendação atual é o EVE-NG Community. Os motivos:

  • Interface web moderna — acesse de qualquer dispositivo sem instalar cliente
  • Suporte nativo a mais tipos de imagem (QEMU, IOU, Dynamips, Docker)
  • Topologias colaborativas — compartilhe o acesso ao lab com colegas
  • Mais estável com muitos nós simultâneos comparado ao GNS3
  • Adotado pela maioria dos cursos de CCNP e CCIE no Brasil

O GNS3 ainda é excelente escolha se você prefere open source puro, usa Mac ou Linux como sistema principal, ou precisa da integração nativa com Wireshark para análise profunda de protocolos.

Recomendação por nível

Iniciante (zero experiência): comece com Cisco Packet Tracer — é gratuito, roda em qualquer PC e os recursos do NetAcad são excelentes para fundamentos. Intermediário (CCNA/CCNP): migre para EVE-NG Community com imagens IOSv. Avançado (CCIE/labs de produção): EVE-NG Professional ou CML com imagens de alta fidelidade e topologias complexas de múltiplos fabricantes.
